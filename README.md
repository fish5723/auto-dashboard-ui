## 📋 项目概述

用 STM32F407 核心板 + 3.5寸 TFT 彩屏做的一套汽车仪表盘 UI，能显示转速、车速、温度、电量，模拟了 CAN 通信数据。

**做这个的原因**：想试试把 FreeRTOS、LVGL、CAN 这些学过的技术串起来做个完整的东西，顺便熟悉一下汽车电子相关的开发流程。

---

## 🛠️ 硬件清单

| 部件 | 型号 | 备注 |
|------|------|------|
| 核心板 |  STM32F407VGT6 | 168MHz，192KB RAM，1MB Flash |
| 显示屏 | 3.5寸 TFT (HX8357DN) | 480×320，FSMC 8080 并口 |
| 下载器 | CMSIS-DAP | 调试用 |
| 电源 | USB 5V 供电  |

---
### 任务划分

| 任务 | 功能 | 优先级 | 周期 |
|------|------|--------|------|
| `task_ui` | 调用 `lv_timer_handler()` 刷新界面 | 高 | 20ms |
| `task_can` | 模拟 CAN 数据帧，更新全局变量 | 中 | 50ms |
| `task_sensor` | 温度、电量等慢变信号采集 | 低 | 100ms |

> TIM7 给 LVGL 提供 1ms 心跳，和 FreeRTOS 的 SysTick 分开，避免互相干扰。
 ---

## ✨ 实现的功能

- [x] FSMC 驱动 HX8357DN，480×320 横屏显示
- [x] LVGL 8.3.11 移植，自定义仪表盘 UI
- [x] 指针式转速表（0-8000 RPM）+ 数字显示
- [x] 指针式时速表（0-240 km/h）+ 数字显示
- [x] 温度条 + 电量条，带颜色变化
- [x] 状态指示灯（转向、故障、安全带等）
- [x] PNG 图标替换默认色块
- [x] FreeRTOS 多任务调度
- [x] **模拟 CAN 通信**（软件生成数据，非真实 CAN 总线）

---

## 🚧 已知局限 & 下一步

**当前不足**：
- CAN 是模拟的，没有接真实 CAN 收发器（TJA1050）
- 没接真实传感器，数据都是代码生成的
- 没做低功耗，USB 直插供电

**想改进的**：
- [ ] 加上 TJA1050 CAN 收发器，接真实 CAN 数据（比如用另一块板子发）
- [ ] 用 DMA2D 加速图形渲染（F407 没有，换 F429/F767 试试）
- [ ] 加上开机自检和故障码显示
- [ ] 学习 DBC 文件，做标准 CAN 信号解析
